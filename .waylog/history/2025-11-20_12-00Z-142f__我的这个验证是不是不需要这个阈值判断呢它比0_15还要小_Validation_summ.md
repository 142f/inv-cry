GitHub Copilot: 常用指标（视频伪造检测）
- Accuracy (ACC)：阈值依赖，受类别不平衡影响大。  
- Precision / Recall / F1：考虑误报/漏报平衡，常用于阈值固定场景。  
- Average Precision (AP，或 mAP)：PR 曲线下面积，适合正负样本不平衡时衡量检索/检测质量。  
- AUC-ROC：ROC 曲线下面积，衡量模型排序能力（不同阈值下的总体表现）。  
- EER（Equal Error Rate）：假正率=假负率时的错误率，常用于二分类认证场景。  
- TPR@FPR（或 FPR@TPR）：在实际运行点（如 FPR=0.01/0.05/0.1）下的查准/查全能力，工程上非常重要。  
- 混淆矩阵 / per-video vs per-frame 指标：视频级聚合（对每个视频取平均/投票）比逐帧指标更贴近实际部署。

哪个更重要
- 不要只看 ACC：当正负样本不平衡时，ACC 容易误导。  
- AP vs AUC：
  - AP（PR-AUC）更适合样本不平衡且关心“高精度”时，强调在高召回下保持精度。  
  - AUC（ROC-AUC）衡量排序能力，适合评估总体判别力。  
- 实际部署时，通常优先报告 AP + AUC，再补充 TPR@低FPR（或 FPR@TPR）与 EER。若必须只选一个，且数据严重不平衡或更关心误报成本，选 AP；若想要整体排序能力，选 AUC。

建议（实践）
- 同时报告：AP、AUC、TPR@FPR（例如 FPR=0.01/0.05/0.1）、EER，并给出视频级指标与混淆矩阵。  
- 把阈值相关指标（ACC、F1）标注对应阈值/采样方式，避免误解。  

结论：AP 与 AUC 都重要，AP 在不平衡和对精度敏感的场景通常更有参考价值；同时务必提供低 FPR 下的 TPR（或固定 FPR 的阈值表现）以满足工程需求。
