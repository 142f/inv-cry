# 多品种海龟趋势系统可行性说明

## 结论

当前项目适合作为 XAU、BTC 及其他高流动性品种的趋势跟踪研究框架，但不应直接当作实盘执行系统。策略核心已经具备突破入场、反向突破出场、N/ATR 波动率归一化、0.5N 加仓、2N 止损、品种簇风险预算、杠杆预算、成本和滑点估算等要素。实盘仍需要补齐真实订单状态、交易所/券商合约约束、延迟、部分成交、断线重连、风控熔断和监控告警。

## 当前实现边界

- 信号采用已完成 K 线确认，订单按下一根 K 线开盘价成交。
- N 使用经典 Wilder 平滑，不使用当前 K 线自身高低点作为突破阈值。
- 回测器会在初始化阶段校验 OHLC 合法性并预计算指标。
- 多品种不同交易日历下，已有持仓用最近一根可用收盘价盯市；新订单只在当前品种有当前 K 线且存在下一根 K 线时生成。
- 盘中止损由 K 线 high/low 触发，跳空按开盘价相对止损价的不利方向成交。
- 成本模型只覆盖固定 bps 成本和滑点，不能替代真实盘口冲击模型。

## XAU 与 BTC 的参数方向

XAU 更接近传统趋势跟踪品种，但会受到美元利率、实际收益率、央行、地缘政治和流动性时段切换影响。BTC 趋势弹性强，但假突破、周末流动性、永续合约 funding 和交易所风险更突出。

建议从以下保守配置开始研究：

- XAU：保留 20/55 入场、10/20 出场作为基线，单品种杠杆上限不超过 1.0，单次 2N 止损风险低于权益的 0.6% 至 1.0%。
- BTC：减少加仓层数，降低单次风险和杠杆，上限建议先控制在 XAU 的 40% 至 60%，必须把 funding、周末滑点和交易所费率纳入回测。
- H4 周期：不要把 20/55 根 H4 直接等同 20/55 天；需要分别测试 `classic-bars` 与 `h4-daily-equivalent`。

## 实盘前必须补充

- 用交易所或券商真实可交易价格重放，而不是只用中间价或净价。
- 加入订单生命周期：提交、确认、部分成交、撤单失败、拒单、滑点超限。
- 加入数据质量守卫：断档、重复 K 线、时区错位、异常 spread、成交量异常。
- 加入组合相关性风控：XAU 与 BTC 在危机行情中可能同时去杠杆，不能只看单品种 N 风险。
- 加入盘前/周末/节假日规则：黄金有休市窗口，BTC 7x24，两者混合回测必须避免交易日历偏差。
